# linux-security-baseline-audit-sample

## Overview
This project demonstrates foundational Linux security practices by performing a baseline security audit of a Linux system.

## Objective
To identify common security weaknesses related to users, permissions, and running services, and document findings in a structured manner.

## Scope
- User and group configuration review
- File and directory permission checks
- Review of running services
- Basic system hardening considerations

## Tools & Skills
- Linux command line
- Bash scripting (basic)
- Security documentation

## Methodology
1. Review system users and groups
2. Identify files with overly permissive permissions
3. Identify unnecessary or risky services
4. Document findings and recommendations

## Deliverables
- Audit checklist
- Sample findings report
- Supporting scripts

## Business Relevance
Linux systems are widely used in enterprise environments. Performing baseline audits helps reduce attack surface and supports compliance efforts.
